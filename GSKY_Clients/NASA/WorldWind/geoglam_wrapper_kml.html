What's This?

This page will create the JS and KML files required to display the GEOGLAM layers in NASA World Wind. Both temporal and spatial dimensions can be specified. It is possible to overlay the GSKY layers with other terrain features like water, scrub, forest, wetland, etc.

The objective of this web app is to display the GSKY layers with other data so that it may make more sense than seeing the GSKY layer alone. For example, the GEOGLAM vegetative cover may be more meaningful if overlaid with forests, grassland, sand, water bodies, etc.

QGIS provides this functionality through 'QuickOSM', but somewhat complex manual settings are required. In this app we are trying to make it a smooth process of just selecting a few params and let the app do the work.

The usefulness of this process must be ascertained before presenting it to the end users.

It is designed to combine the power of GSKY with NASA World Wind (NWW) and uses the data from Overpass for the overlays. This web app is unique in its capability.

It uses Keyhole Markup Language (KML) which was originally developed for Google Earth, and called Keyhole Earth Viewer, to bind together the various layers. Unlike TerriaMap, ArcGIS or QGIS, the NWW does not have a native WMS facility. The KML simulates this functionality to call GSKY.

NWW provides the engine to display the globe, which in turn comes from Bing, and layers.

The overlays are third party data coming from the OpenStreetMap organisation using the APIs to overpass.

Purpose

GSKY provides a WMS interface to display the satellite and geospatial data in both temporal and spatial dimensions. Such data may convey more meaning if combined with other data. In this app, the GEOGLAM vegetative cover can be overlaid with forests, grassland, sand, water, etc. so that the display may make more sense.
 

How to Run:

  • Choose the required GEOGLAM layer.
  • Choose the data acquisition date.
  • Choose a listed region, enter a BBox or place the crosshair.
  • Choose one or more overlays as key/value pairs.

Fine Details:

  • Use the BBox Finder to draw a bounding box and click the paste () button.
  • Or, locate the crosshair at a required spot and click the crosshair () button.
    • The dropdown defines the BBox limits (units of 0.2° for X and Y)
  • The World Wind pane will open to show the globe.
    • Click one of the links on the navigation pane on left to go to the layers.
  • The GSKY layer plus the overlay(s) will be shown on the map.
    • You can turn on/off any layer in the navigation pane.
    • Note: multiple overlays on a large area may freeze the World Wind engine.

Details

Many GIS clients such as TerriaMap, ArcGIS, QGIS, NASA WorldWind and Google Earth Desktop (GE) support the WMS services. They request the Capability statement via a GSKY server URL. Thereafter the "GetMap" requests are based on the visible Lat/Lon coordinates on the screen.

GEWeb does not have a WMS support as above. However, it can send the GetMap requests via a KML file. This allows us to manually request the data for any region by specifying the coordinates and the date of data acquisition. The latter is very useful, as all other clients except TerriaMap can only show the latest time slice.

While GE works almost similar to TerriaMap, it lacks the ability to display time layers as in TerriaMap. When using their WMS service there is no way to specify a time layer as e.g. TIME=2001-01-01.

Here we provide a way to create functional KML files where the time and Lat/Lon coordinates are coded via web-based controls. Such KML files can be opened in GEWeb. (Note: Currently the KMLs are not compatible with GE, but we are working on it!)

You will see a GSKY layer for each KML file imported. The layers will stay permanently and hence you do not have to recreate the files every time.

You will, however, need to create a new KML file to see another time layer or another bounding box. Unlike in the WMS service, you cannot simply pan to a different part of the globe and get the layers re-displayed as you move.

You can zoom in/out, and the GSKY layers will get smaller/bigger but their resolution will not change. To see a smaller region, select from the "Geographical Region" dropdown or specify your own Lat/Lon values.

Notes:
  • GEWeb currently works only in Google Chrome.
  • The selectable geographical regions available are limited to Australa, NZ, USA and Europe.
    • We may add more regions in the future.
    • You can manually specify the Lat/Lon coordinates of any region.
    • If you specify a large region it may not show the layers, but will show "Zoom in to view". However, zooming in will not bring up the layers.
